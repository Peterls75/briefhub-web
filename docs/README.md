# BriefHub documentation

Living documentation for the BriefHub build, versioned with the codebase.

## Now

- [`reference/homepage.pdf`](reference/homepage.pdf) — the canonical homepage design. The look and feel of every other page calibrates against this.

## Planned (named here so they have a designated home)

- `refinement-criteria.md` — the criteria the AI scope refinement runs against. Phase 4. Provided by Peter.
- `refinement-prompt.md` — the prompt that calls those criteria into the model context. Phase 4. Provided by Peter.
- `scope-document-template.html` — the HTML template the BriefHub Scope Document renders from. Used both in-portal and for the PDF. Phase 4.
- `intake-questions.md` — the fourteen questions on the brief intake form, with the conditional logic between them. Phase 3. Provided by Peter.
- `email-templates/` — the transactional email templates. Phases 3, 4, 6, 7, 8. Provided by Peter.
- `specialisms.md` — the controlled vocabulary of practice specialisms used to match briefs to panel lawyers. Provided by Peter.

## Convention

These documents are the source of truth for content. When the AI refinement criteria change, the change is a deliberate reviewable commit. Same for the BriefHub Scope Document template. Live content does not drift from documentation.
